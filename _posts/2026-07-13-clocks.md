Every codebase that touches time eventually faces the same interrogation in code review, which is: how do you even test this? The timeout logic, the retry backoff, the cache expiry, the timing wheel; all of it consults `std::chrono::steady_clock::now()`, and `now()` answers to nobody. So the test suite either sleeps (and your CI pipeline slowly becomes a meditation retreat), or it doesn't test time at all (and the backoff logic ships with an arithmetic bug that only manifests at hour boundaries), or somebody threads a `Clock` template parameter through fourteen layers of code that never wanted to be templates.

Corvid's [`clocks.h`](https://github.com/stevensudit/Corvid/blob/main/corvid/infra/clocks.h) takes a fourth path, and it's 139 lines including the license block. The idea: a single, static indirection point per clock type.

```cpp
template<typename Clock, size_t>
class now_clock {
  inline static std::atomic<now_fnt> now_fn_{&clock_t::now};
public:
  [[nodiscard]] static time_point_t now() noexcept(noexcept(Clock::now())) {
    return now_fn_.load(std::memory_order::relaxed)();
  }
  static void set_now_fn(now_fnt fn) noexcept;
  // ...
};

using steady_now_clock = now_clock<std::chrono::steady_clock, 1>;
```

Production code calls `steady_now_clock::now()` wherever it would have called `steady_clock::now()`. By default that loads a function pointer that points at... `steady_clock::now`, so nothing changes. Tests call `set_now_fn` to install anything they like, and the built-in fake — an atomic `time_point` you set with `set_fake_now` — covers the common case of "make it be 3:47 and then make five minutes pass instantly."

The ergonomic centerpiece is the RAII guard:

```cpp
auto guard = steady_now_clock::fake_now_scope();
steady_now_clock::set_fake_now(t0);
// ... exercise the timeout logic ...
steady_now_clock::set_fake_now(t0 + 30s);
// ... assert the timeout fired ...
// guard's destructor restores the real clock, even on test failure
```

`fake_now_scope` installs the fake and returns a `scope_exit` that reinstates `&clock_t::now` when it dies, so a throwing assertion can't leave the process living in fake time, which is the classic failure mode of set-and-forget clock mocking.

## The anonymous size_t

The template signature is `now_clock<typename Clock, size_t>`, and that unnamed `size_t` is the header's best detail. It exists because of an implementation fact about standard clocks that most people learn the hard way: `std::chrono::high_resolution_clock` is almost always a *typedef* — of `steady_clock` on some implementations, `system_clock` on others. Class templates instantiate on types, and a typedef is not a new type. Without the extra parameter, `now_clock<high_resolution_clock>` and `now_clock<steady_clock>` would be *the same class* on libstdc++, sharing one set of statics, and a test that fakes the high-resolution clock would silently fake the steady clock too, but in a way that differs across standard libraries. The `size_t` is a disambiguation tag, nothing more: each alias gets a distinct number, each instantiation gets its own `now_fn_` and `fake_now_`, and a portability landmine is defused by one anonymous parameter. Seven aliases are provided, covering `steady`, `system`, `file`, `high_resolution`, `utc`, `gps`, and `tai`.

## What the atomics are and aren't doing

Both statics use `memory_order::relaxed`, and the comment explains the reasoning rather than cargo-culting `seq_cst`: the loaded value is a lone function pointer or `time_point` that changes rarely (in practice, only in tests), no other memory is published through it, so no ordering relationship needs establishing and atomicity is only there to prevent torn reads. This is the correct minimal contract, and it also tells you what the design *isn't* promising: this is not a synchronization point, and swapping clocks mid-flight while worker threads race `now()` is a test-orchestration problem the header rightly declines to solve.

There's also a small fossil of compiler archaeology worth pointing at: the function-pointer type is declared as `time_point_t (*)() noexcept(noexcept(Clock::now()))`: a conditionally-noexcept function pointer type, propagating the clock's own exception specification, except on MSVC, where an `#ifdef` drops the `noexcept` because the frontend chokes on the construct. `noexcept` became part of the type system in C++17; compilers are still digesting.

The conversions corner (`as_nanoseconds` / `from_nanoseconds`) exists for the boundary where time leaves C++ — wire formats, epoll timeouts, logging — and it handles the awkward endpoint honestly: `time_point::max()` maps to `UINT64_MAX` and back, so "never" survives a round trip through an integer instead of overflowing into "soon."

## The trade, stated plainly

This design charges production a real, small price for testability: every `now()` is a relaxed atomic load plus an indirect call, where a direct `steady_clock::now()` call could be inlined down to its syscall or `rdtsc` sequence. For almost every consumer — timeouts, expirations, logging — this is noise; the clock read itself dwarfs the indirection. If you're timestamping at a rate where an indirect call registers, you're not the audience for this header, and nothing forces it on you.

What you buy is the absence of the usual alternatives' costs. Against the template-parameter approach (`template<typename Clock> class timeout_sweeper`), you avoid infecting every signature between the clock and its consumers, and you avoid the combinatorial test-instantiation problem. Against dependency injection, you avoid threading a clock object through constructors that otherwise need nothing. Against link-time seams and weak symbols, you keep it portable and visible. The static-global nature is the honest weakness: two tests faking the same clock can't run concurrently in one process. But test frameworks serialize within a binary by default, and `fake_now_scope` at least guarantees they can't *corrupt* each other across sequential runs.

139 lines, one idea, and a disproportionate improvement in what's testable. Apache-2.0 at [`corvid/infra/clocks.h`](https://github.com/stevensudit/Corvid/blob/main/corvid/infra/clocks.h).
